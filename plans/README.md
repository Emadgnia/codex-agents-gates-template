# plans/README.md

Use one plan file per task:

- Naming: `YYYY-MM-DD-short-task-name.md`
- Example: `2026-02-28-open-source-quality-audit.md`

Every task plan should include:
1. Goal and scope
2. Assumptions and missing inputs
3. Execution steps
4. Validation steps and commands
5. Risks and rollback plan
6. Explicit approval status

Approval must be captured before implementation starts.
